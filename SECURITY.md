# Security Policy

## Supported Versions

Lux is currently pre-1.0. Security fixes are provided on a best-effort basis
for the latest release line and `main`.

| Version | Supported |
| --- | --- |
| `main` (unreleased) | yes |
| latest `v0.x.y` release | yes |
| older releases | no |

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Use GitHub private vulnerability reporting for this repository:

1. Go to the repository Security tab.
2. Click "Report a vulnerability".
3. Provide reproduction details, impact, and affected version/tag.

If private reporting is unavailable in your fork/environment, open a draft
security advisory in this repository and request maintainer review.

## Response Targets (Draft)

- Acknowledgement: within 3 business days.
- Initial triage: within 7 business days.
- Mitigation/fix plan: as soon as severity is confirmed.

## Disclosure Policy

Lux follows coordinated disclosure:

- Do not publicly disclose details until maintainers confirm a fix is available
  or explicitly approve disclosure.
- Maintainers will publish release notes for security fixes and request CVE IDs
  when appropriate.
